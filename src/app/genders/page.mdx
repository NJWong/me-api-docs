# Genders

Genders for characters in the Mass Effect universe. {{ className: 'lead' }}

## The gender model

The gender model is very simple.

### Properties

<Properties>
  <Property name="id" type="number">
    Unique ID for the gender.
  </Property>
  <Property name="name" type="string">
    The gender name.
  </Property>
</Properties>

---

## List all genders {{ tag: 'GET', label: '/api/v1/genders' }}

<Row>
  <Col>

    This endpoint returns a paginated list of all genders. By default, a maximum of ten results are shown per page.

    ### Optional attributes

    <Properties>
      <Property name="limit" type="integer">
        Limit the number of genders returned. Defaults to 10 if not specified.
      </Property>
      <Property name="offset" type="integer">
        Offset the results by skipping the specified number of genders. Defaults to 0 if not specified.
      </Property>
    </Properties>

    <div className="not-prose">
      <Button
        href="/pagination"
        variant="text"
        arrow="right"
        children="Read the docs for pagination"
      />
    </div>

  </Col>
  <Col sticky>

    <CodeGroup title="Request" tag="GET" label="/api/v1/genders">

    ```bash {{ title: 'cURL' }}
    curl -G https://me-api.njwon4.workers.dev/api/v1/genders -d limit=10
    ```

    ```js {{ title: 'JavaScript (fetch)' }}
    const response = await fetch('https://me-api.njwon4.workers.dev/api/v1/genders?limit=10')
    ```

    </CodeGroup>

    ```json {{ title: 'Response' }}
    {
      "meta": {
        "limit": 10,
        "offset": 0,
        "total": 3
      },
      "data": [
        {
          "id": 1,
          "name": "Male",
        },
        {
          "id": 2,
          "name": "Female",
        },
        {
          "id": 3,
          "name": "Non-binary",
        }
      ]
    }
    ```

  </Col>
</Row>
